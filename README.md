## Overview

This portfolio documents a structured exploration of prompt engineering techniques using Claude, a large language model by Anthropic. Through a series of real-world tasks — from explaining financial concepts to children, to drafting professional emails, to testing for hallucinations and bias — this project demonstrates how the quality of a prompt directly shapes the quality of an AI's output.

The work covers prompt design and iteration, parameter control, strategic prompting methods, and responsible AI evaluation, with every decision documented and explained.

---

## Portfolio Structure

### Part 1: Prompt Design & Iteration
Three practical scenarios, each taken through three prompt versions (V1 → V3), with assessments explaining what changed and why the output improved.

- **Scenario A — Technical Explanation:** Explaining compound interest to a 10-year-old
- **Scenario B — Professional Email:** Cold outreach internship email for Kotak Mahindra Bank
- **Scenario C — Creative Content:** Marketing tagline for a sustainable fintech startup (Greenvault)

Includes a comparison table for each scenario and an analysis of how role assignment and context setting affect model outputs.

### Part 2: Temperature & Parameter Control
A controlled experiment using an identical base prompt at three temperature levels (low, medium, high), with a detailed comparison table and recommendations for when each setting is most appropriate.

### Part 3: Strategic Prompting Techniques
- **Chain-of-Thought Prompting:** Applied to a multi-step financial ratio problem, comparing outputs with and without step-by-step reasoning instructions
- **Few-Shot Prompting:** Financial news sentiment classification tested zero-shot vs. few-shot, with analysis of when examples add the most value

### Part 4: Responsible AI & Limitations
- **Hallucination Testing:** Two fabricated research prompts used to probe the model's honesty and uncertainty-handling
- **Bias Testing:** Comparative analysis of how the model frames high-status vs. customer-facing roles, and how prompts can correct occupational bias
- **Limitations & Recommendations:** Three key limitations identified and three actionable recommendations for responsible LLM use

---

## Key Learnings

- **Role and context transform outputs** — assigning a persona (e.g. "friendly primary school teacher") shifts vocabulary, tone, and structure more effectively than instruction alone
- **Chain-of-thought prompting improves auditability** — showing reasoning step-by-step makes outputs verifiable, which is critical for financial and academic tasks
- **Explicit honesty instructions reduce hallucination risk** — prompts that say "say so if you don't know" produce more transparent and trustworthy responses
- **Bias can be subtle** — even gender-neutral language can carry occupational bias through the qualities and vocabulary chosen to describe different roles
